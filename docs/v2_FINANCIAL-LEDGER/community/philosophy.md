# 프로젝트 철학

## 핵심 가치

### 1. 완전 무료

> "모든 기능은 영원히 무료입니다."

- ❌ 프리미엄 기능 없음
- ❌ 유료 모듈 없음
- ❌ 기능 제한 없음
- ❌ 사용자 수 제한 없음
- ❌ 광고 없음

**이유:**
이 프로젝트는 서비스가 아닌 도구입니다. 돈이 아닌 자유를 추구합니다.

### 2. Self-hosted 우선

> "당신의 데이터는 당신의 것입니다."

- ✅ 사용자가 직접 서버 운영
- ✅ 모든 데이터는 사용자 서버에
- ✅ 중앙 서버 없음
- ✅ 데이터 수집 없음

**이유:**
개인정보 보호는 타협할 수 없는 가치입니다.

### 3. 오픈소스

> "투명성이 신뢰를 만듭니다."

- ✅ 전체 소스 공개 (MIT License)
- ✅ 개발 과정 공개
- ✅ 이슈 트래킹 공개
- ✅ 로드맵 공개

**이유:**
숨길 것이 없고, 함께 만들어가는 것이 더 좋은 결과를 만듭니다.

### 4. 커뮤니티 중심

> "혼자보다 함께가 더 강합니다."

- ✅ 커뮤니티 의견 적극 수용
- ✅ 함께 의사결정
- ✅ 모든 기여를 환영
- ✅ 투명한 거버넌스

**이유:**
이 프로젝트는 한 사람이 아닌 커뮤니티가 만듭니다.

## 설계 철학

### 1. 서비스가 아닌 프레임워크

**이것은 SaaS가 아닙니다:**
- 호스팅 서비스 제공 ❌
- 중앙 서버 운영 ❌
- 구독 요금 ❌

**이것은 프레임워크입니다:**
- 사용자가 직접 배포 ✅
- 사용자가 직접 제어 ✅
- 완전한 자유 ✅

### 2. 모듈 중심 아키텍처

**왜 모듈 시스템인가?**

- **확장성**: 필요한 기능만 설치
- **유연성**: 원하는 대로 커스터마이징
- **커뮤니티**: 누구나 모듈 개발 가능
- **지속가능성**: 핵심은 안정적으로, 기능은 자유롭게

**WordPress 처럼:**
- 핵심은 작고 안정적
- 플러그인으로 확장
- 생태계 중심

**VSCode 처럼:**
- 확장 기능으로 강력해짐
- 개발자 친화적
- 빠른 혁신

### 3. 개인정보 보호 우선

**설계 원칙:**
- 수집하지 않는 것이 최선의 보안
- 사용자가 모든 것을 제어
- 투명한 데이터 흐름

**구체적 실천:**
- DB는 사용자 서버에
- AI API Key는 사용자가 직접 관리
- 통합 서비스 인증은 사용자가 설정
- 프로젝트는 어떤 개인정보도 수집하지 않음

## 개발 철학

### 1. 완벽보다 진행

> "Done is better than perfect."

- 빠르게 출시, 빠르게 개선
- MVP부터 시작
- 지속적인 반복

### 2. 문서화 우선

> "코드는 한 번, 문서는 천 번 읽힙니다."

- 코드만큼 중요한 문서
- 예제 중심
- 초보자 친화적

### 3. 안정성 > 기능

> "새 기능보다 안정성이 우선입니다."

- Core는 최소 변경
- 하위 호환성 유지
- 충분한 테스트

### 4. 개발자 경험

> "좋은 DX가 좋은 UX를 만듭니다."

- 명확한 API
- 일관된 패턴
- 풍부한 문서
- 쉬운 개발 환경

## 비즈니스 철학

### 1. 상업화하지 않습니다

**왜?**

상업화는 다음을 가져옵니다:
- 투자자 압력
- 수익 우선
- 기능 제한
- 프리미엄 모델
- 광고

우리는 이 모든 것을 거부합니다.

### 2. 지속 가능성

**어떻게 유지하나?**

- 선택적 후원
- 커뮤니티 기여
- 최소한의 인프라 비용
- 자발적 개발

**목표:**
- 돈이 아닌 가치 추구
- 장기적 비전
- 커뮤니티 중심

### 3. 경쟁 대신 협력

**다른 프로젝트와:**
- 아이디어 공유
- 코드 공유
- 함께 성장

**상업 제품과:**
- 대체가 아닌 선택지
- 각자의 영역
- 상호 보완

## 커뮤니티 철학

### 1. 모든 기여는 가치있습니다

**코드만이 전부가 아닙니다:**
- 버그 리포트
- 문서 개선
- 질문에 답변
- 홍보
- 피드백

모두 소중한 기여입니다.

### 2. 환영하는 문화

**우리가 추구하는 것:**
- 친절함
- 인내심
- 포용성
- 존중

**우리가 거부하는 것:**
- 엘리트주의
- 무례함
- 배타성
- 차별

### 3. 투명한 운영

**모든 것을 공개합니다:**
- 개발 과정
- 의사결정
- 재정 상황
- 문제점

**이유:**
신뢰는 투명성에서 나옵니다.
